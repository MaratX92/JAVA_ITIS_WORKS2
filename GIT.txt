1) Скачать Git-клиент
https://git-scm.com/downloads -> Git Bash
2) Регистрируемся https://github.com/
3) Переход во вкладку Repositories
4) Нажимаете New
5) Вводите название репозитория JAVA_ITIS_WORKS2
6) Нажать Create Repository
Сейчас вы создали удаленный репозиторий
7)В блоке Quick Setup в строке HTTPS скопировать содержимое вида https://github.com/{ваше имя}/JAVA_ITIS_WORKS2.git
8) Открываете Git Bush
cd Desktop - (Change Directory) - сменить директорию
ls - список файлов в директории
ls -A показать все файлы, включая скрытые
9) Клонируем удаленный репозиторий к себе на компьютер
git clone https://github.com/{ваше имя}/JAVA_ITIS_WORKS2.git
10) Создать папки с названиями TaskНомерЗадачи и переносите свои коды
11) git status - команда вывода информации о текущем статусе локального репозитория.
Варианты:
Красное название папки/файла - значит, что файл только что был добавлен и никак не отслеживается git-ом (они не в треке файлов)
12) git add название_файла_папки - добавляет файл в трек, гит начинает за ним следить
После выполнения этой команды, в случае добавления этого файла только что, будет написано new file
Теперь, если файл будет изменен, то после вызова git status вам будет об этом сообщено
13) git diff показывает разницу между файлом до вызова add и после вызова add
14) Делаем коммит изменений (слепок) git commit -m 'какое-нибудь сообщение'
15) git add . - добавляет сразу все файлы в трек
16) git log - показывает список коммитов
17) git reset 2704e749524415bf825e6ee579c0b545e4f19dee
 - отмена коммитов, следующих после указанного коммита
 18) Заливаем изменения в удаленный репозиторий:
 git push origin master
 19) В браузере обновляете страницу с репозиторием JAVA_ITIS_WORKS2
 20) git pull - если хотим изменения из удаленного репозитория
 21) Если скомпилировать .java-файл, то получится так, что создастся бинарный class-файл, который git не умеет отслеживать. И вообще, загружать исполняемые файлы не нужно по понятным причинам.
 22) файл .gitignore - нужен для того, чтобы дать понять системе git, что отслеживать некоторые файлы не нужно.

АДРЕСА GIT-а скинуть
